# Implementation Plan: BGS Check Export — Health Data Check & Export Tool

**Branch**: `001-check-export` | **Date**: 2026-05-16 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/001-check-export/spec.md`

## Summary

BGS Check Export is a web-based tool for hospital staff to search, filter, and review health records
from the Thai Ministry of Public Health 43 standard data files (43 แฟ้มมาตรฐาน), then export the
results as a complete Excel (.xlsx) file for reporting and submission.

Architecture: TypeScript monorepo (Express REST API backend + React SPA frontend). The backend
queries the existing EHP CIS database (read-only), applies filters, and streams Excel generation.
The frontend provides a search form, paginated results table, detail modal, and export button.
No new database is introduced — the system reads from the existing hospital database.

## Technical Context

**Language/Version**: TypeScript 5.x (Node.js 20 LTS backend; React 18 frontend)

**Primary Dependencies**:
- Backend: Express 5, exceljs (Excel generation), zod (validation), cors, dotenv
- Frontend: React 18, React Query 5, React Hook Form, Vite 5, Tailwind CSS, axios

**Storage**: Existing EHP CIS database (read-only connection via environment config)

**Testing**: Jest + Supertest (backend); Jest + React Testing Library (frontend)

**Target Platform**: Node.js 20 LTS server; Web browser (Chrome/Edge on Windows)

**Project Type**: Web application — monorepo with Express API backend + React SPA frontend

**Performance Goals**: Search results in <5s for 1-month range; Excel export of 10k rows in <60s

**Constraints**: Read-only database access; no mobile; no auth complexity (single-hospital LAN deployment)

**Scale/Scope**: ~10 concurrent users; up to 50,000 records/search; single-site hospital deployment
